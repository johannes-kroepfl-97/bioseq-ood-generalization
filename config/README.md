# Configuration

Every run in `pipeline_phases.py` is parameterized from three authored files plus
one generated folder. There is exactly one home for each kind of setting.

| File | Authored / generated | Holds |
|------|----------------------|-------|
| `training.yaml` | authored | Shared training defaults (lr, epochs, batch size, schedulers, …) used by **every** run, plus the architecture-independent `search_space` and the `mlflow` on/off toggle. Edit a training default here once and all models/datasets/methods pick it up. |
| `models.yaml` | authored | One block per encoder. `defaults` = the `model.*` architecture config; `search_space` = the architecture grid swept by the Phase B encoder search. |
| `methods.yaml` | authored | Per-method **static defaults**, deep-merged under `training` for missing keys only. Parameter-free methods (`erm`, `adabn`) are empty blocks. |
| `tuned/models.yaml` | **generated** (Phase B) | `dataset → model → tuned encoder architecture` — the winning `model.*` block per dataset. |
| `tuned/methods.yaml` | **generated** (Phase E) | `dataset → model → method → tuned hyperparameters`. |

Mental model: **`tuned/X.yaml` is the resolved version of `X.yaml`** — the search
picks a winner from the space declared in the authored file and writes it to
`tuned/`.

## What is *not* here

- **Method search distributions** (log-uniform draws, conditional ranges) live in
  `src/bioseq_ood/methods/hparams.py`, not YAML — they are sampling logic, not
  data. `methods.yaml` holds only the concrete defaults.
- **Study-wide constants** — the honest-selection rule (`selection.mode = val_id`),
  the legacy `setting` field, and the per-protocol adapt/test pools — are injected
  in code by `pipeline_phases.py` (`_apply_study_constants`, `PROTOCOLS`). They are
  fixed methodological choices, not per-run knobs, so they are deliberately absent
  from the YAML.

## Editing rules

- Change a shared training value → `training.yaml`.
- Change/inspect an architecture or its search grid → `models.yaml`.
- Change a method default → `methods.yaml`.
- **Do not hand-edit `tuned/`** — it is regenerated by the search phases. It is
  committed to git so final-protocol runs reproduce without re-searching; delete a
  file there to force that phase to search again on the next run (`RESUME` reads
  `tuned/` when present).

## How a run config is assembled

`base_cfg(model, dataset)` = `training.yaml` (minus `search_space`)
`+ models.yaml[model].defaults` `+ {dataset.name}`, then `_apply_study_constants`
pins the methodological constants and `apply_method_hparams` fills method defaults.
The Phase B encoder search merges `training.yaml[search_space]` with
`models.yaml[model].search_space` via `search_space_for(model)`.
